存储器部分

  1. RAM 地址寄存器：存储了 RAM 中数据的地址（包括 RAM 和 AUX-RAM），CPU 在需要读写 RAM 时候告诉 CPU 数据的存取位置
  2. RAM：更靠近 CPU，存放局部变量、寄存器的值等
  3. *AUX-RAM：“物理上片内，逻辑上外部”，扩展内存，是辅助 RAM 存放更多数据的
  1. 程序存储器（Flash）：永久==保存程序、固件等大块数据==，断电不丢失，MCU 启动后从这里运行
  2. EEPROM：用于保存需要长期存储的值，断电不丢失，主要==储存经常需要修改的少量数据==，如==配置参数、用户设置、校准数据等==

[!notes] 区别

  • Flash 一般成块（Block）或者页（Page）写入，速度快，写入寿命比下面的短，但容量大
  • EEPROM 支持按字节写入，适合修改少量数据，速度慢，写入寿命比上面的长，但容量小
  • 后记：实际上目前国产STC宏晶系列不管是Flash还是EEPROM都全用上Flash芯片了，也不是特别分这个，实际不是MCU仙人，平常做项目根本没有要写进这里面的，相当于是断电还要保存的数据，要断电重启的项目应该也开始用stm32之类的了，51还是太简单了（逃）

运算控制部分

  1. ACC（累加器）
    • 本质是一个寄存器，用来==存储==运算过程中的数据和最终结果
    • 叫累加器是==因为经常在累加过程中用到==，当然==加减法逻辑运算都支持==
  2. **临时寄存器（TMP1 & TMP2）
    • 用于保存中间数据
    • 两个可能是因为方便复杂运算（第一步结果存到 1，第二步存到 2）
  3. ALU（算术逻辑单元）
    • ==执行==所有算术和逻辑运算
    • 从累加器和临时寄存器去除操作数，计算，存回累加器
  4. PSW（程序状态字寄存器）
    • 作用： 主要用来==记录计算状态==，记录的是标志位，如：
      • 进位标志（CY）： 如果运算结果超过了寄存器的存储能力（比如 8 位运算结果超过 255），就会设置进位标志。
      • 零标志（Z）： 如果运算结果是 0，这个位会被置位。
      • 溢出标志（OV）： 如果运算结果超出了允许的范围（比如符号位有问题），会设置这个标志。 记录的是重要事件，相当于备忘录，“师傅，该进一了”
  1. 双数据指针
    • 允许MCU同时指向两个不同的内存地址，可以快速切换数据（左脚踩右脚）
    • 比如：一个读数据，一个写数据；左右交替处理数组之类的
  2. 堆栈指针
    • 就像一个==书签==，标识/记录堆栈的顶部位置
    • 堆栈是一个特殊的内存区域，用来管理程序调用，比如函数之间的跳转和返回
    • 该指针可以在：
      • 调用一个函数时，保存当前的位置，以便函数执行完毕后可以正确返回
      • 嵌套函数调用中，“找对正确的路”

定时通信部分

  1. 定时器 0/1 与定时器 2
    • 精确管理时间/计数，如：时间控制、时间计数
    • 大概的模式：
      • 定时模式：根据时钟源计时，达到设定时间后触发一个中断
      • 计数模式：根据外部输入信号脉冲个数计数，达到几几次后触发中断
    • 多个定时器便于同时管理多个任务
    • 0/1 和 2 分开==可能是==因为 0 与 1 设计差不多，而与 2 不一样
  2. 串口（UART）
    • MCU 与其他设备通信的桥梁，使用串行通信协议，一位一位地发送信息
    • 特点
      • 单通道：MCU 一次只能与一个设备通信（用协议可以在外拓展）
      • 速度调节：支持不同波特率（传输速度）
      • 双向通信：允许双向发送消息
  3. 看门狗定时器（WDT）
    • 安全机制，程序正常运行时，MCU 会定期重置 WDT；==当程序卡住/死循环==，看门狗没有被重置，WDT 会触发==复位==，重新启动 MCU

程序控制 & 存储器交互部分

  1. PC（程序计数器）
    • 作用：是一个寄存器，用来储存当前正在执行的指令的地址，引导执行下一条指令
    • 工作原理：
      • 每执行一条指令，PC 的值自动增加，指向下一个指令的地址
      • 遇到跳转指令时候（JUMP 或者 CALL 等），PC 会被修改为跳转目标的地址
  2. 地址生成器
    • 作用： 地址生成器负责生成 MCU 内部各模块所需的存储器地址。
      • 包括程序存储器（Flash）、数据存储器（RAM）和特殊功能寄存器（SFR）的地址。
    • 用途：
      • 当处理器需要从存储器读取数据或写入数据时，地址生成器会根据指令提供对应的地址。
      • 它与程序计数器配合，确保指令和数据在正确的时间被正确访问。

[!Question] 为什么 PC 和地址生成器都要的是“地址”，而感觉好像这俩地址不一样？ 因为==PC==中的地址是逻辑地址，是抽象地址 而真正的物理地址是由地址生成器处理的，它将 PC 的逻辑地址查找/计算出对应物理地址 因此实际的时序是：当处理器准备取下一条指令时候，PC 的值已经更新，递给地址生成器，其根据这个值生成对应 Flash 中地址

  1. Flash（程序存储器）
    • 作用：一般用来存储程序，可反复擦写，运行时只读，编程时可以通过 ISP/IAP 擦写
  2. ISP/IAP（在线编程与应用编程）
    • ISP：通过特定接口（USB、串口）修改 Flash 内容
    • IAP：程序运行时，更新 Flash 中的部分内容（比如 OTA）
  1. 存储器和控制模块的协作
    • PC 指向当前指令的地址
    • 地址生成器根据 PC 提供的地址，从程序存储器中读取指令（实际上是 Flash 读取指令，生成器只是生成了地址）
    • 指令被送到控制单元，控制单元解析指令并产生对应的操作
    • 如果需要访问数据存储器（RAM）或特殊寄存器，地址生成器会生成相应的地址
    • 执行完成后，PC 自动指向下一条指令，进入下一个循环

I/O 接口

  1. 锁存器(Port 0,1,2,3,4 锁存器)

    • 功能：
      • 保存和稳定数据。用于保存 I/O 口的数据状态，比如，当 CPU 将一个数据写入到某个 I/O 端口时，锁存器负责将数据固定，直到被下一个数据覆盖
      • 双向的。可以把==输出==的信号稳定，也可以把==输入==的信号去抖/同步/……
    • 工作方式：
      • CPU 输出数据时，数据会经过总线，先进入锁存器，锁存器会将其稳定后再传递给 I/O 驱动器
      • 锁存器会暂时存储从总线上传输的数据，并将其锁定（Latch）。直到信号完全稳定后，才会将数据传递给后续模块（如 I/O 驱动器）。
      • 锁存器通过时钟信号或者控制信号同步数据切换，确保输出的信号是一个完整、确定的值。
    • 提供了 MCU 内部与外部的缓冲区和隔离层

  2. 驱动器

    • 功能
      • 负责将锁存器输出的数据==传出==物理引脚
      • ==接收==外部信号，通过驱动器传递到 CPU 通常有电平转换电路，用于适应不同的外部设备电压要求

  3. I/O 引脚（P0, P1, P2, P3, P4）

    • 功能
      • 不同的端口支持不同的功能，例如某些引脚可以配置为 UART、SPI、I2C 或 ADC 通道。
    • 数据流
      • 当 CPU 要输出数据时：数据 → 锁存器 → 驱动器 → I/O 引脚。
      • 当 CPU 要读取数据时：外部信号 → I/O 引脚 → 驱动器 → CPU。

控制单元

  • 功能：协调模块运作，解析指令，产生控制信号
  • 组成：
    1. 时钟电路：XTAL1 与 XTAL2 是外部晶振接口，提供一个固定频率的时钟信号
    2. 复位引脚：重新加载程序计数器、清空寄存器，确保系统从头开始运行
  • 作用：
    1. 从存储器中取出指令
    2. 解析指令内容
    3. 生成操作信号

总线

  • 传递数据、地址和控制信号的通道
  • 分类：
    1. 数据总线
    2. 地址总线
    3. 控制总线
  • 总线是一种共享资源。为了避免多个模块同时使用总线导致冲突，MCU 内部会有一个总线仲裁器（Bus Arbiter）来管理总线的访问权。
