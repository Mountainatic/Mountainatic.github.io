如何提交一个高质量的Issue (Bug报告)

您不应该提一个“Question”，而应该直接提一个“Bug report”（Bug报告），因为我们已经明确了问题所在和解决方法。

  1. 访问OpenWrt的GitHub Issues页面: https://github.com/openwrt/openwrt/issues

  2. 点击绿色的 “New issue” 按钮，然后选择 “Bug report”。

  3. 按照模板填写报告内容。为了让开发者能最高效地看懂，我为您准备好了一份高质量的报告模板，您只需要复制、粘贴并稍作修改即可。

Bug报告模板 (您可以直接使用)

Title (标题): [v23.05.3][mediatek/filogic] Docker package fails to compile due to incorrect build output path

Body (内容):

Environment (环境)

  • OpenWrt Version: v23.05.3 (stable release)

  • Target: mediatek/filogic

  • Target Profile: MediaTek MT7981 rfb

  • Host System: WSL2 with Docker on Windows using the official ghcr.io/openwrt/sdk container.

Steps to Reproduce (复现步骤)

  1. Clone the v23.05.3 source code.

  2. Initialize feeds: ./scripts/feeds update -a && ./scripts/feeds install -a

  3. Run make menuconfig, select the mediatek/filogic target and MT7981 rfb profile.

  4. Select the docker package from Utilities.

  5. Run the compile command: make package/docker/compile V=s

Actual Behavior (实际行为) The compilation fails with the following error, indicating the final binary cannot be found by the install script:

Expected Behavior (预期行为) The docker package should compile successfully.

Analysis (问题分析) The verbose build log shows that the upstream build script (./scripts/build/binary) inside the Docker source code incorrectly outputs the compiled binary to a hardcoded path, instead of the path expected by the OpenWrt Makefile.

  • Actual Output Path: $(PKG_BUILD_DIR)/x86/64/docker-linux-arm64

  • Expected Path by Installer: $(PKG_BUILD_DIR)/build/docker

Workaround / Fix (修复方案) The issue can be resolved by modifying the Package/docker/install definition in feeds/packages/utils/docker/Makefile to fetch the binary from the correct (but wrongly named) path and rename it.
