[!Question]+ 为什么需要这么多关键字划分 RAM？

  • 8051 内部 RAM 主要分成三部分：

    1. 直接寻址区—— data 直接寻址最快，==受 CPU 指令集限制，只能直接访问 128 字节的 data 区==
    2. 间接寻址区—— idata 通过访问寄存器 R0 或 R1 进行间接寻址
    3. 特殊功能寄存器区（SFR） 这里的地址和 idata 的高 128 字节重叠，但通过不同的寻址方式区分（直接寻址访问 SFR，间接寻址访问 idata 的高 128 字节）

    ==也就是什么呢，idata 区在 data 区前，data 是 前 128 Bytes，后 128 Bytes 叫高 128 Bytes，间接寻址访问，直接寻址后 128 Bytes 访问到的是 SFR==

    0x00 - 0x7F 是 data 区，0x80 - 0xFF 表为 SFR，里是 idata 区

[!Question]+ 说了半天，R0 和 R1 是哪里的寄存器？ 实际上，片内 RAM 的 data 区可以更加细分：

  • 工作寄存器区（低 32 Bytes，0x30 - 0x7F）

    • 每组 8 个寄存器，共有 4 组，分别存储在地址范围 0x00 ~ 0x1F
    • ==寄存器按组启用，每次只能启用一个完整的寄存器组==，具体由 PSW 的 RS1 和 RS0 决定 因你而在的故事#运算控制部分 RS1 = 0, RS0 = 0：启用组 0（地址范围 0x00 ~ 0x07） RS1 = 0, RS0 = 1：启用组 1（地址范围 0x08 ~ 0x0F） RS1 = 1, RS0 = 0：启用组 2（地址范围 0x10 ~ 0x17） RS1 = 1, RS0 = 1：启用组 3（地址范围 0x18 ~ 0x1F）

    R0和R1这些只是每回启用的组中的寄存器的代号罢了，叫 R0-R7 ==当然是有特殊情况的，多任务切换或者中断嵌套调用中可能会用多个寄存器组==

    • 除了正在被当作工作寄存器使用的组外，其他都可以当普通 RAM 使用
    • 动态切换寄存器组时，其他组的内容会保留。
    • 可以通过地址直接访问非激活组中的寄存器：

    unsigned char data *ptr; //定义指针变量 unsigned char temp; ptr = (unsigned char *) 0x08; //指针指向组 1 起始地址 *ptr = 0x55; temp = *(ptr+1); //读取组 1 的第二个寄存器内容 P1 = temp; //输出到 P1 口```

  • 可位寻址区（128位，0x20 - 0x2F）

    • 一般来说用作存储布尔型变量、标志位等纯逻辑变量，按位读取写入
    • 必要时也可存储普通变量，按字节读取写入
    • 定向引用得要汇编/位操作

  • 普通 RAM 区（低 80 Bytes，0x30 - 0x7F）

    • 局部变量、全局变量、堆栈指针

  • 特殊变量类型

    1. sbit 特殊位变量 用于访问 SFR 或者位地址，详细的说是 ==将位地址绑定到一个变量==

      绑定到位寻址区：sbit varible = adresss^number 绑定到 SFR：sbit varible = SFR名字^位序号

    2. sfr 特殊功能寄存器 用于将 SFR 中的寄存器地址绑定到变量（比如把 0x90 绑定到 P1） 一般现代 C51 编译器已经预制，==无需重复定义== 只能绑定 0x80 - 0xFF（也就是 SFR 内部） sfr 只能定义 1 Byte 的寄存器 用 SFR 定义的表示完整八位的地址的变量，只能按字节读写，如需逐位操作需要结合 sbit 使用

    3. 操纵各位电平 例如：P1 = 0xFF 将 P1 的所有引脚设置为高电平 ==每个字节是打包设置各个位的== 0xFF 是 16 进制数，转换为二进制就是 1111 1111

[!Question]+ 什么是位地址？

  • 位地址允许对单个==位==进行访问和操作
  • 每个位地址唯一标识 RAM 或 SFR 的某一具体位，范围为 0x00 ~ 0x7F，即支持 128 个位地址
  • 分为两类：
    1. 位寻址区中的位地址（0x20 - 0x2F）->（0x00 - 0x7F）
    2. SFR 中的位地址（0x80 - 0xFF）
      • 但是某些 SFR 位置支持直接访问
  • 也就是什么呢，位地址把字节地址又细分，给每个位都分配了地址，8 个位地址长度=1 个正常地址，也就是所谓的==映射==

[!Question]+ 那 sbit 和 bit 在位寻址区有啥区别？ 你要想清楚，sbit 是把那个==位==绑定到变量进行操作，bit 是一个位变量，被分配到了位寻址区，这两个建立联系过程是相反的，并且 bit 不一定会分配到特定的位上

[!Question]+ 那位地址有啥用？没看见用啊？ 位地址在汇编中才能用，C 代码那种编译时候也会转成汇编用位地址写 如汇编的啥 SETB、CLR、JB , label、JNB ， label 在汇编文档中还会有 0**H 这种写法，和 0x** 是等同的，16 进制数
